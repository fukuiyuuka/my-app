■以下は完了している前提
・githubアカウント・リポジトリ作成
・githubにアップロード
    ・backendは、ローカルでnpm run build(tsc)してdistもアップロードすること
        ・production環境だとnpm installでdevdependenciesに入っているパッケージはインストールされないので、npm run build(tsc)でエラーになる。
    ・frontendは、よくわからないがrender側でbuildしても問題なかった。（vue-tscが上手くやってくれている？）
・Render登録（githubと連携）
・Upstash登録（githubと連携）

■DBサーバ
    ・RenderでPostgreSQL作成
    ・後でバックエンドサーバでURLを設定し、マイグレーションもさせる。

■Redisサーバ
    ・UpstashでRedisサーバ作成

■バックエンドサーバ
    ・RenderでWeb Service作成
        ・RunTime：Node.js
        ・Build Command：npm install && npx prisma generate && npx prisma migrate deploy
            ・npm run build（tsc）はローカルで実施する。
        ・Start Command：npm run start
            ※npm run start = node dist/server.js
        ・環境変数 
            ・DATABASE_URL
            ・REDIS_HOST
            ・REDIS_PORT
            ・REDIS_PASSWORD
            ・FRONT_ORIGIN

■フロントサーバ
    ・RenderでStatic Site作成
        ・Build Command： npm run build
        ・Publish Directory: dist
        ・環境変数…Web Serviceのように指定はできないので、ファイルで分ける。
            ・VITE_API_URL: バックエンドサーバのURL
                ・.env.local：ローカル実行時のURLを記載。npm run dev(vite)実行時に読み込まれる。gitアップロード不要。
                ・.env.production：本番実行時のURLを記載。npm run build(vite build)実行時に読み込まれる。gitアップロードが必要。
